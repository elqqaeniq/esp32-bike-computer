3. Notes & warnings

QSPI advantage — 4 parallel data lines @ 40 MHz give ≈4× the bandwidth of single-SPI on the same clock. A full 360×360 RGB565 frame (≈ 252 KB) takes ~13 ms at 40 MHz QSPI vs ~50 ms at 40 MHz single-SPI.
QSPI ≠ SPI flash — ESP32-S3 OPI PSRAM uses dedicated flash QSPI lines (GPIO 26–37). Display QSPI is a separate logical bus on regular GPIOs (10/11/12/13/15/38). Don't confuse them.
If breadboard wiring is flaky — drop TFT_SPI_FREQ from 40 MHz to 20 MHz in config.h. 4 parallel lines on long jumper wires can cross-talk and corrupt frames. PCB with proper layout handles 40 MHz fine.
If colors are inverted — flip the 4th argument of Arduino_ST77916(tftBus, RST, ROT, true) from true to false. IPS panels vary.
3.3V only on display — ST77916 module is NOT 5V tolerant on logic or supply. Both VCC and IO lines must be 3.3V — same as ESP32-S3 GPIOs.
TE pin behavior (v1.1.2) — drives the FPS-counter ISR only. Display draw loop is NOT yet sync'd to TE — that's deferred to v1.2 with partial redraws. If TE wire is missing, ISR never fires and gDisplayFps stays 0; no other harm.
Partition table change (v1.1.2) — when upgrading from v1.1.1, run one upload with Erase All Flash Before Sketch Upload → Enabled, otherwise the bootloader keeps the old layout. After that upload set it back to Disabled to preserve NVS.
